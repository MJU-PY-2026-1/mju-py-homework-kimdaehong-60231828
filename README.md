# 🐍 [2026-01] 나만의 파이썬 소프트웨어 개발 프로젝트

## 1. 시나리오 제목
군대 체력 측정 관리 시스템
<br>


## 2. 시나리오 (5~10줄)
한달에 한번 정기적으로 시행하는 체력측정의 결과를 용사별로 정리하고 용사의 체력 수준을 파악할 수 있는 시스템이다. 3종목의 체력 측정의 결과를 입력하면 결과에 따른 등급(특급,1,2,3,불합격)이 나오고 3종목을 합산하여 종합 등급을 산출한다. 중대원들의 결과를 순위대로 나열할 수 있고 특급이 나온 인원에게는 특급전사 칭호를 부여하고 4박 5일의 포상 휴가를 부여한다.
<br>


## 3. 예상 기능 및 메뉴 (최소 5개)
    1. 신규 용사 등록
    2. 종목별 점수 환산 및 종합 체력 계산
    3. 체력 등급 판정
    4. 특급전사 여부 판정
    5. 포상휴가 부여(특급:4박5일)
체력 측정 종목은 3km달리기, 팔굽혀펴기, 윗몸 일으키기가 있고
달리기부터 12분30초 이내 - 특급, 이후로 1분 단위로 1급,2급,3급,
팔굽혀펴기는 72개 이상 특급, 이후로 8개 단위로 1,2,3급(ex.64-71 1급)
윗몸일으키기는 86개 이상 특급, 이후 8개 단위로 1,2,3급(ex.78-85 1급)
종합 체력 환산할때 3종목 all 특급이어야만 특급으로 환산해야함
<br>

# 🚀 [버전별 개발 일지 & AI 협업 기록]

## 🟦 [1차 과제: V1.0] 시나리오 기획
    
### **🤖 AI 파트너십 과정**
내용1: 파이썬 기초 한계에 맞춘 현실적인 1단계 기획 도출
    프롬프트 요약: 아직 함수나 반복문을 배우지 않은 파이썬 입문 수준에서 군대 체력측정 관리 프로그램을 어떻게 만들어야 할까?
    적용 내용: AI와 논의하여 현재 학습 수준(변수, 조건문)에서 수십 명의 데이터를 다루는 것은 비효율적임을 인지함. 이를 바탕으로 1차 기획의 목표를 무리하게 잡지 않고, '1명의 데이터를 입력받아 즉시 등급     과 포상휴가를 판독하는 기능'만 넣은 뒤 반복문 등 다른 기능들을 학습 한 뒤 새로은 기능들을 추가하기로 확립함
    
내용 2 : 군대 체력측정 기준의 논리적 코드화 및 데이터 전처리 아이디어 획득

적용 내용: 사용자가 입력하는 3km 달리기 기록(분, 초)을 그대로 비교하면 로직이 복잡해지므로, AI의 제안을 받아들여 입력받은 분(minute)을 초(second) 단위로 통합 환산(예: 12분 30초 = 750초)하는 전처리 방식을 도입함. 또한 종합 등급 판정 시 or 연산자를 활용하여 '3가지 종목 중 하나라도 불합격/3급/2급/1급이 있는지' 순차적으로 걸러내어, 최종적으로 아무 조건에도 걸리지 않은 인원만 '특급'으로 분류되게 하는 효율적인 판정 알고리즘을 설계함.

프롬프트 요약: "달리기 12분 30초 이내 특급, 팔굽혀펴기 72개 이상 특급 등 종목별 기준이 있고, 종합 체력은 3종목 모두 특급이어야만 해. 이걸 어떻게 코드로 짤까?"

## 🟩 [2차 과제: V1.0] 입출력 + 리스트 + 조건문 - 향후 작성 예정
### **✨2차 과제 내용:**
- 이번 2차 과제는 기존 단일 인원 체력 측정 기능을 확장하여 중대 특급전사 관리 시스템을 구축하는 것입니다
- 여러 장병의 데이터(이름, 계급, 뜀걸음, 팔굽혀펴기, 윗몸 일읔키기)를 input()으로 반복해서 입력받아 list에 저장합니다
- 입력 과정에서 for 반복문을 순회하며 각 장병의 체력 데이터를 수집하고, if~elif~else 조건문과 and/or 논리 연산자를 활용하여 특급전사, 1급, 2급, 3급, 불합격 등급을 자동으로 판정합니다. 'or' 논리 연산자를 활용해 가장 낮은 등급이 종합 등급이 되도록 구현했으며, 중첩 'if'문과 'and' 연산자를 더해 압도적인 기록을 낸 인원에게 '최우수 특급전사' 타이틀을 부여하는 로직을 추가했습니다.
- 수집된 데이터 리스트를 바탕으로 부대 총 팔굽혀펴기 횟수를 누적 계산(+=)하고, max(), len(), append, sort 등의 내장 함수를 활용하여 중대 종합 통계를 도출하는 지능적 로직을 완성했습니다.
- 
### **🤖 AI 파트너십 과정**
(예시) 
1. **내용 1: 데이터 구조 설계 자문**
    * **프롬프트 요약:** "여러 명의 이름과 점수를 리스트 하나에 담아서 관리하고 싶은데 가장 효율적인 구조가 뭘까?
    * **적용 내용:** 각 장병의 정보를 리스트로 묶어 다시 전체 리스트('soldiers'에 담는 '2차원 리스트' 구조를 채택하여 데이터 관리의 효율성을 높임.

2. **내용 2: 엄격한 군대식 종합 등급 로직 구현**
    * **프롬프트 요약:** "체력측정은 3종목 중 하나라도 낮은 등급이 있으면 그 등급이 종합 등급으로 결정 되는걸 코드로 짜고싶어."
    * **적용 내용:** 복잡하게 최소값을 찾는 대신, 'if' 문과 'or' 연산자를 활용해 불합격이 하나라도 있는가?->3급이 있는가? -> 2급이 있는가? 순으로 차례대로 필터링하는 직관적인 로직을 완성함
    
### **🛠️ Troubleshooting & 기술 회고:**
(예시) 
1. **문제 1: (TypeError)**
    * **원인:** `input()'으로 받은 뜀걸음 시간을 형변환하지않고 계산에 사용해 오류 발생. .
    * **해결:** `소수점기록(예: 12.30)을 처리하기 위해 'float()를 사용해 해결함.
2. **문제 2:  (Indentation Error)**
    * **원인:** for 반복문 안에 if 문을 작성할때 들여쓰기 깊이가 맞지 않아 실행 오류 발생
    * **해결:** 파이썬의 핵심인 블록(Block) 구조를 다시 복습하며 일정한 간격으로 띄어쓰기를 수정함.
